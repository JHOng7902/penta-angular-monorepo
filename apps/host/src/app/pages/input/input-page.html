Input Component

The shared pt-ui-input supports flexible types, labels, icons, validation lengths, clear action, and two-way binding.

Import & Use

  1. Import InputComponent from the shared library.
  2. Add it to the imports array of your standalone component.
  3. Bind using [(value)] or [value] + (valueChange).
import { InputComponent } from '@penta/shared-ui';

@Component({
  standalone: true,
  imports: [InputComponent],
})
export class ExampleComponent {}

<pt-ui-input label="Name" [(value)]="name"></pt-ui-input>

Password Preview

Use allowPasswordPreview to show a toggle that reveals/hides password values. Works alongside the clear button. 

<pt-ui-input
  type="password"
  [allowPasswordPreview]="true"
  [showClear]="true"
></pt-ui-input>

Input Types

Supported types: text, email, password, number, tel, url, search, date, datetime-local, time. Radio/checkbox/button are intentionally blocked. 

<pt-ui-input label="Email" type="email"></pt-ui-input>
<pt-ui-input label="Password" type="password"></pt-ui-input>

Date, Date-Time, Time (Malaysia)

Date returns just the selected date (YYYY-MM-DD). Date-time and time emit ISO 8601 with a Z suffix (UTC-style), keeping the exact local time the user picked.

Payload values: Date: {{ dateValue || '—' }} | Date-Time: {{ dateTimeValue || '—' }} | Time: {{ timeValue || '—' }} 
<pt-ui-input label="Date" type="date" [(value)]="dateValue"></pt-ui-input>
<pt-ui-input label="Date-Time" type="datetime-local" [(value)]="dateTimeValue"></pt-ui-input>
<pt-ui-input label="Time" type="time" [(value)]="timeValue"></pt-ui-input>

Label Positions

Control label placement with labelPosition: top, left, or hidden. 

<pt-ui-input label="Left Label" labelPosition="left"></pt-ui-input>
<pt-ui-input label="Hidden Label" labelPosition="hidden" placeholder="..."></pt-ui-input>

Required + Validation Length

Use required, minLength, and maxLength to guide validation. Required fields show a red asterisk. 

<pt-ui-input label="Required Field" [required]="true"></pt-ui-input>
<pt-ui-input [minLength]="3" [maxLength]="12"></pt-ui-input>

Disabled State

Bind disabled to any condition. Toggle below to see it update. 

<pt-ui-input [disabled]="isLocked"></pt-ui-input>

Leading Icon

Use icon to show a leading label or symbol inside the input. If the text matches a Material icon name (ex: search), it renders the icon. If not, the text is shown as-is. Set iconPosition to right and enable iconClickable to make the icon clickable and listen to (iconClick). 

<pt-ui-input label="Email" icon="email"></pt-ui-input>
<pt-ui-input label="Search" icon="search"></pt-ui-input>
<pt-ui-input label="Search" icon="search" iconPosition="right"></pt-ui-input>

Right Icon + Key Events

Make the right icon clickable with iconClickable, and listen for keyupEvent / keydownEnterEvent.

Events: Icon clicks: {{ iconClickCount }} | Last key: {{ lastKey || '—' }} | Enter count: {{ enterCount }} 
<pt-ui-input
  icon="search"
  iconPosition="right"
  [iconClickable]="true"
  (iconClick)="onInputIconClick()"
  (keyupEvent)="onInputKeyup($event)"
  (keydownEnterEvent)="onInputEnter($event)"
></pt-ui-input>

Clear Button

Enable showClear to show a clear icon when text is present. 

<pt-ui-input [showClear]="true"></pt-ui-input>

Blinking Attention

Use blink to draw attention to an input (optional). 

<pt-ui-input [blink]="true"></pt-ui-input>

Textarea

Set multiline to render a textarea with the same styling and bindings. 

<pt-ui-input label="Description" [multiline]="true"></pt-ui-input>

Two-Way Binding

Use [(value)] to keep the UI and data in sync.

Current value: {{ basicValue || '—' }} 
<pt-ui-input [(value)]="name"></pt-ui-input>

Inputs & Outputs

Inputs

  • label: string
  • type: string (text, email, password, number, date, time, datetime-local)
  • placeholder: string
  • value: string
  • required: boolean
  • disabled: boolean
  • labelPosition: top | left | hidden
  • minLength: number
  • maxLength: number
  • icon: string
  • iconPosition: left | right
  • iconMode: auto | text
  • iconClickable: boolean
  • showClear: boolean
  • multiline: boolean
  • allowPasswordPreview: boolean
  • blink: boolean
  • id: string
  • name: string

Outputs

  • valueChange: emits current string
  • iconClick: emits when the right icon is clicked
  • keyupEvent: emits keyboard events from the input
  • keydownEnterEvent: emits when Enter is pressed
