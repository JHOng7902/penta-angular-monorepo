Input Component

The shared pt-ui-input supports flexible types, labels, icons, validation lengths, clear action, and two-way binding.

Import & Use

  1. Import InputComponent from the shared library.
  2. Add it to the imports array of your standalone component.
  3. Bind using [(value)] or [value] + (valueChange).
import { InputComponent } from '@penta/shared-ui';

@Component({
  standalone: true,
  imports: [InputComponent],
})
export class ExampleComponent {}

<pt-ui-input label="Name" [(value)]="name"></pt-ui-input>

Password Preview

Use allowPasswordPreview to show a toggle that reveals/hides password values. Works alongside the clear button. 

<pt-ui-input
  type="password"
  [allowPasswordPreview]="true"
  [showClear]="true"
></pt-ui-input>

Input Types

Supported types: text, email, password, number, tel, url, search, date, datetime-local, time. Radio/checkbox/button are intentionally blocked. 

<pt-ui-input label="Email" type="email"></pt-ui-input>
<pt-ui-input label="Password" type="password"></pt-ui-input>

Date, Date-Time, Time (Malaysia)

Date returns just the selected date (YYYY-MM-DD). Date-time and time emit ISO 8601 with a Z suffix (UTC-style), keeping the exact local time the user picked.

Payload values: Date: {{ dateValue || '—' }} | Date-Time: {{ dateTimeValue || '—' }} | Time: {{ timeValue || '—' }} 
<pt-ui-input label="Date" type="date" [(value)]="dateValue"></pt-ui-input>
<pt-ui-input label="Date-Time" type="datetime-local" [(value)]="dateTimeValue"></pt-ui-input>
<pt-ui-input label="Time" type="time" [(value)]="timeValue"></pt-ui-input>

Label Positions

Control label placement with labelPosition: top, left, or hidden. 

<pt-ui-input label="Left Label" labelPosition="left"></pt-ui-input>
<pt-ui-input label="Hidden Label" labelPosition="hidden" placeholder="..."></pt-ui-input>

Required + Validation Length

Use required, minLength, and maxLength to guide validation. Required fields show a red asterisk. 

<pt-ui-input label="Required Field" [required]="true"></pt-ui-input>
<pt-ui-input [minLength]="3" [maxLength]="12"></pt-ui-input>

Disabled State

Bind disabled to any condition. Toggle below to see it update. 

<pt-ui-input [disabled]="isLocked"></pt-ui-input>

Leading Icon

Use icon to show a leading label or symbol inside the input. If the text matches a Material icon name (ex: search), it renders the icon. If not, the text is shown as-is. Set iconPosition to right and enable iconClickable to make the icon clickable and listen to (iconClick). 

<pt-ui-input label="Email" icon="email"></pt-ui-input>
<pt-ui-input label="Search" icon="search"></pt-ui-input>
<pt-ui-input label="Search" icon="search" iconPosition="right"></pt-ui-input>

Right Icon + Key Events

Make the right icon clickable with iconClickable, and listen for keyupEvent / keydownEnterEvent.

Events: Icon clicks: {{ iconClickCount }} | Last key: {{ lastKey || '—' }} | Enter count: {{ enterCount }} 
<pt-ui-input
  icon="search"
  iconPosition="right"
  [iconClickable]="true"
  (iconClick)="onInputIconClick()"
  (keyupEvent)="onInputKeyup($event)"
  (keydownEnterEvent)="onInputEnter($event)"
></pt-ui-input>

Scan Input

Enable scanMode to submit after a debounce or on Enter. The input auto-clears on submit, can auto-focus, and supports max-length limits.

Scan result: {{ scanResult || 'â€”' }} | Count: {{ scanCount }} | Current value: {{ scanValue || 'â€”' }} 
<pt-ui-input
  [scanMode]="true"
  [scanAutoFocus]="true"
  [scanDebounceMs]="500"
  [scanSubmitOnEnter]="true"
  [scanClearOnSubmit]="true"
  [scanMaxLength]="12"
  (scanSubmit)="onScanSubmit($event)"
></pt-ui-input>

Clear Button

Enable showClear to show a clear icon when text is present. 

<pt-ui-input [showClear]="true"></pt-ui-input>

Blinking Attention

Use blink to draw attention to an input (optional). 

<pt-ui-input [blink]="true"></pt-ui-input>

Textarea

Set multiline to render a textarea with the same styling and bindings. 

<pt-ui-input label="Description" [multiline]="true"></pt-ui-input>

Two-Way Binding

Use [(value)] to keep the UI and data in sync.

Current value: {{ basicValue || '—' }} 
<pt-ui-input [(value)]="name"></pt-ui-input>

Customize Styles

Wrap inputs in a class and override styles locally. For component-scoped styles, use ::ng-deep or move overrides to a global stylesheet. 

<div class="input-page__custom-theme">
  <pt-ui-input label="Branded Input"></pt-ui-input>
</div>

:host ::ng-deep .input-page__custom-theme .pt-ui-input__label {
  color: #4338ca;
}

:host ::ng-deep .input-page__custom-theme .pt-ui-input__field input,
:host ::ng-deep .input-page__custom-theme .pt-ui-input__field textarea {
  border-color: #a5b4fc !important;
  background: #eef2ff !important;
  border-radius: 12px;
  color: #1e1b4b;
}

:host ::ng-deep .input-page__custom-theme .pt-ui-input__field input:focus,
:host ::ng-deep .input-page__custom-theme .pt-ui-input__field textarea:focus {
  border-color: #6366f1 !important;
  box-shadow: 0 0 0 3px rgba(99, 102, 241, 0.2) !important;
}

Inputs & Outputs

Inputs

Name Type Usage
labelstringLabel text shown above/left of the input.
typestringInput type (text, email, password, number, date, time, datetime-local).
placeholderstringPlaceholder text shown when empty.
valuestringBind the current value.
requiredbooleanShows a required indicator and native required attribute.
disabledbooleanDisables the input.
labelPositiontop | left | hiddenControls label placement.
minLengthnumberMinimum length hint/validation.
maxLengthnumberMaximum length hint/validation.
scanModebooleanEnable scan behavior (debounce/Enter submit).
scanAutoFocusbooleanAuto-focus when scan mode is active.
scanDebounceMsnumberDelay before auto-submit after typing stops.
scanSubmitOnEnterbooleanSubmit scan when Enter is pressed.
scanClearOnSubmitbooleanClear input after scan submit.
scanMaxLengthnumberMaximum characters accepted during scan.
iconstringMaterial icon name or text label shown inside the field.
iconPositionleft | rightPlace the icon on the left or right.
iconModeauto | textForce text icon or auto-detect material icons.
iconClickablebooleanEnable click handling for right icon.
showClearbooleanShow a clear button when value is present.
multilinebooleanRender a textarea instead of input.
allowPasswordPreviewbooleanShow/hide password toggle for password type.
blinkbooleanEnable attention blink animation.
idstringCustom input id attribute.
namestringCustom input name attribute.

Outputs

Name Type Usage
valueChangeEventEmitter<string>Emits the current value on input.
iconClickEventEmitter<void>Emits when the right icon is clicked.
keyupEventEventEmitter<KeyboardEvent>Emits keyup events from the input.
keydownEnterEventEventEmitter<KeyboardEvent>Emits when Enter is pressed.
scanSubmitEventEmitter<string>Emits the scan payload when submitted.
