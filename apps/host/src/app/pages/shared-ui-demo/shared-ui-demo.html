Shared UI Library

Reusable Angular components for the Penta platform

This library provides consistent UI building blocks (buttons, inputs, tables, cards, toasts) used across apps in this monorepo. Use it to keep styles aligned and accelerate feature delivery.

Shared UI v0.0.1
Angular ~21.1.0
Node >= 20.19.0
Nx 22.4.2
TypeScript ~5.9.2

Requirements

  • Node >= 20.19.0 (see .nvmrc)
  • Angular ~21.1.0 with Angular Material + CDK
  • RxJS ~7.8.0

Install (Monorepo)

Install dependencies from the repo root, then run the host app:

npm install
npx nx serve host

Build the shared UI package when needed:

npx nx build shared-ui

Import & Use Components

Import components directly from @penta/shared-ui:

import { Button, InputComponent, Table } from '@penta/shared-ui';
<pt-ui-button buttonText="Save"></pt-ui-button>
<pt-ui-input label="Email" type="email" [(value)]="email"></pt-ui-input>
<pt-ui-table [data]="rows"></pt-ui-table>

Component prefix: pt-ui-

Create A New App (Standalone)

Generate a standalone app and start importing shared components:

npx nx g @nx/angular:app my-app --standalone --style=scss
npx nx serve my-app

Create A New App (NgModule / Non-Standalone)

Create a module-based app and import shared components into the module:

npx nx g @nx/angular:app my-legacy-app --standalone=false --style=scss
// app.module.ts
import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { Button, InputComponent, Table } from '@penta/shared-ui';

@NgModule({
  declarations: [AppComponent],
  imports: [BrowserModule, Button, InputComponent, Table],
  bootstrap: [AppComponent],
})
export class AppModule {}
